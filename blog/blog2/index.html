Problem 52 - Permuted multiples

  • Problem Description:

  • Solution:

  • Within the nested function check_digits, we have two functions. The first function is is_same, is_same is used to check if two number have the same length, and also to check if they contain exactly the same digits (could be in different orders). One thing to be noticed in this function is that, say we have two numbers, num1 and num2. We may first pick out the first digit in num1 and see if it’s in num2, and if it does, it’s very important to make a note in num2, specifically note the digit in num2 that was used to match the digit in num1 such that it can’t be used anymore. The second function is give_ans, give_ans is just a helper function to help us quicker get the final result. It compares the input number and 2,3,4,5,6 times itself. If it returns 1, then the input number satisfy the problem requirement.

  • Starting with input number of 1, check if the input number satisfy the problem requirement, with increment of 1, until it meets the requirement. Finally, it ends at 142857.

Problem 112 - Bouncy numbers

  • Problem Description:

  • Solution:

  • Within the nested function is_bouncy, we have three functions. They are check_increase, check_decrease and check_bouncy respectively. Apparently, they are each used to check if the input number is they type that they each is responsible for. For check_increase and check_decrease, they simply first convert the type of input integer into string, then comparing the value of each digit of the string from left to right and see if it continuously (increase or stay the same) or (decrease or stay the same). Both functions just mentioned return 1 if the input number belongs to the type they each is responsible for and return -1 if the opposite. For function check_bouncy, it simply check whether the number is neither increasing nor decreasing using the functions check_increase and check_decrease.

  • start from 100 (as we know that all two digits numbers must not be bouncy numbers), we check if the number is bouncy, with increment of 1, until the count of bouncy numbers / total count of numbers is exactly 0.99. Note that within every loop, nums is incremented by 1, and bouncy_nums only increment if the input number pass the is_bouncy’s check (in this case, return 1).
